# AI写作与字数规范

这份文档用于统一本项目后续由 AI 生成文本时的长度、重点和输出格式。  
原则很简单：**不同文件，不同字数，不同职责**。

## 1. 通用规则

- 所有文本默认使用中文。
- 这里的“字数”按中文字符量理解，允许上下浮动 15%。
- 生成内容必须围绕本项目现有设定，不要引入新世界观冲突。
- 每个文件只负责一件事，不要把剧情、设定、台词、说明全塞进同一份文本。
- 需要细节，不要只写梗概。
- 不要写“总结式说明”，尽量写成可直接拆进游戏脚本的内容。

## 3. 统一输出格式

建议所有 AI 生成文档都遵循这个结构：

```text

用途：

必须包含：
风格要求：
禁止事项：
输出格式：
```

```text
- 必须包含：【开场 / 冲突 / 转折 / 选项点 / 章节结尾】
- 重点：【人物互动 / 情绪变化 / 画面感 / 分支钩子】
- 禁止：【纯梗概 / 跳叙太快 / 一口气讲完设定 / 没有对白】

- 只输出正文
- 允许按小节分段
- 对白要像 galgame，不要像说明书
```

## 6. 分支选择写法建议

- 一个分支文件只保留一条主冲突。
- 如果同一章节有多个选项，优先写“结果差异”，不要把所有结果都写成一样。
- 每个选择都要带来至少一种变化：
  - 关系变化
  - 情绪变化
  - 信息变化
  - 场景变化
